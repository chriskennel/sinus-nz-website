# Project: ENT & Allergy Website

## Overview
A single-page Astro + Tailwind CSS website for Dr. Chris Kennel, FRACS ENT surgeon in Hastings, Hawke's Bay, NZ. Deployed on Netlify.

## Domains
- **Primary:** `entallergy.nz` (and `www.entallergy.nz`)
- `sinus.nz` → redirects to `entallergy.nz/#nose` (via Netlify redirect in `netlify.toml`)
- `entallergy.co.nz`, `enta.nz`, `allergyhb.nz`, `allergyhb.com` → all redirect to `entallergy.nz` (via 1stDomains URL Redirection)
- Domain registrar: **1stDomains** (1stdomains.nz)
- DNS for `entallergy.nz`: A record `75.2.60.5` (Netlify load balancer), CNAME `www` → `entallergy.nz`

## Hosting
- **Netlify** (paid personal plan)
- GitHub repo: `chriskennel/sinus-nz-website`
- Netlify auto-deploys on every push to `main`
- Each production deploy costs **15 Netlify credits** — batch changes into single pushes to conserve credits (300/month on free plan)
- Build command: `npm run build`, publish dir: `dist`

## Key Files
- `src/pages/index.astro` — entire site content (single page)
- `src/components/Hero.astro` — hero section with JS parallax
- `src/components/Nav.astro` — navigation with mobile hamburger menu
- `src/layouts/BaseLayout.astro` — HTML head, sticky CTA button, meta tags
- `netlify.toml` — build config, security headers, and redirect rules (sinus.nz → entallergy.nz/#nose)
- `public/robots.txt` — allows all crawlers, points to sitemap
- `astro.config.mjs` — site URL set to `https://entallergy.nz`, includes sitemap integration

## Redirects
All redirect rules live in `netlify.toml` — there is no redirect UI in Netlify dashboard. Edit that file to change redirects.

## Forms
- Consultation request form uses **Netlify Forms** (`data-netlify="true"`)
- Form name: `consultation-request`
- On submission, redirects to `/success` (thank you page at `src/pages/success.astro`)
- Email notifications configured in Netlify dashboard: Site → Forms → Notifications
- Spam protection via honeypot field (`bot-field`)

## SEO
- Sitemap auto-generated by `@astrojs/sitemap` at `https://entallergy.nz/sitemap-index.xml`
- Submitted to **Google Search Console** and **Bing Webmaster Tools**
- Bing verification meta tag in `BaseLayout.astro`: `msvalidate.01`
- Canonical URL set to `https://entallergy.nz/`

## Security Headers (netlify.toml)
- Content-Security-Policy (allows Google Fonts and Google Maps iframe)
- X-Frame-Options: DENY
- X-Content-Type-Options: nosniff
- Referrer-Policy: strict-origin-when-cross-origin

## Design Decisions (from prior session)
- Hero image is intentionally **black & white** — distinctive against colour-heavy competitor sites; green CTA button pops more against monochromatic background
- Condition category cards use David sculpture JPG images with black backgrounds (not transparent PNGs) — black provides more visual weight, prevents washed-out look
- Logo in nav uses CSS `invert` filter to make black SVG appear white on dark nav — no separate white logo file needed
- Logo is a trademark registered in New Zealand — no ™/® symbol added to site (not legally required)
- Nav logo file: `ENTA logo solid tragus_Black on transparent large text.svg` with `class="h-14 w-auto invert"`
- Nose condition image uses `object-[center_75%]` crop (not default center — too much top; not bottom — too much lip)
- FAQ section appears before Request/Consult section in page order (matches nav order)
- `scroll-margin-top: 64px` applied to all `section[id]` to offset fixed nav header
- Unused working files (PSDs, alternate headshots, raw images) remain in project root — not served, just not cleaned up

## Known Issues / Design Decisions
- `bg-fixed` (CSS parallax) disabled on mobile due to iOS Safari bug — JS parallax used instead (in `Hero.astro`)
- Hamburger menu uses CSS peer checkbox pattern — checkbox must be a direct sibling of the mobile menu div
- Sticky CTA uses `env(safe-area-inset-bottom)` to clear iOS address bar
- Sticky CTA hides when hero section OR request section is visible (threshold: 0)
- Google Maps iframe requires `frame-src https://www.google.com` in CSP
